# 0. 全体の開発方針

- **まずは動く最小機能（MVP）を作る**
- 複雑な機能（ブランチ/マージなど）は **バックログに逃がす**
- コードはまず自分で書いて、**AIに添削させる運用**
- ドキュメントは「生きたドキュメント」として随時更新
- 一度に全部やらず、**ユースケース単位で刻む**

# 6. AI の使い方（プロジェクト運用ルール）

## 6-1. あなたの役割

- 設計を書く（要件・ユースケース・DB）
- コードの第一稿を書く
- エラーと詰まった点を相談する

## 6-2. ChatGPT の役割

- 設計のレビュー＆修正案
- バグの原因説明
- コードの改善点指摘
- 書いたコードのリファクタ提案

## 6-3. Codex / Cursor の使い方

- ボイラープレート（同じようなコード）
- リファクタリング
- 小さめの補完
- 大きめの仕様は ChatGPT とあなたで設計 → Cursor は補助のみ
